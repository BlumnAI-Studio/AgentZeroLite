# M0026 — 에이전트 악단 유튜브 임베드 무대 (펜슬 설계안)

## 실행 요약
미션 브리프가 **"펜슬로 먼저 디자인 → 검토 → 구현"** 게이트를 명시하여,
구현에 앞서 설계도만 작성했다. 운영자에게 사전 확인 2건을 받았다:
- 메타/카테고리: **oEmbed(키 불필요) + 설정된 LLM 분류** 병행
- 악단 반응: **시스템 루프백 그대로** (별도 배선 없음)

설계도(`Docs/design/M0026-agent-band-youtube.pen`)는 두 부분:
1. **앱 윈도우 목업** — 창 크기 고정, 가운데 스테이지(1fr)만 상/하로 분할.
   - 무대 상단 URL 바(붙여넣기 → [붙이기] → "✓ 메타 획득")
   - TopZone: 영상 플레이어(좌, 캡션에 채널·LLM분류·카테고리 칩) + 재생목록 패널(우 308px, 카테고리 탭 + 항목 선택)
   - BandZone: 악단·연주단 풀폭 유지(리드 보컬 중앙 글로우 + 악기/백업), 라벨 스트립에 "🔊 시스템 루프백 반응"
   - 푸터: 스펙트럼 바 + diag (기존 유지)
2. **설계 노트 보드** — ①레이아웃 ②데이터 흐름(oEmbed+LLM) ③악단 반응 ④검토 포인트(리스크) ⑤구현 범위·수용기준

## 검토 포인트 (구현 전 결정 필요)
1. **LLM 호출 표면** — `zero.chat.send`는 사용자 채팅 히스토리를 오염시킴.
   (권장) 호스트 `WebDevBridge.cs`에 stateless `llm.classify`(또는 `chat.oneshot`) op 1개 추가.
   미도입 시 `chat.reset` 회피책은 사용자 대화를 지우므로 비권장. → **결정 필요**.
2. **WebView2 외부 iframe/네트워크(youtube.com) 허용** — CSP·가상호스트(zero.local) 경유 확인 필요.
3. **분류 실패/오프라인 폴백** — 제목 키워드 휴리스틱 → 최후 '기타'.
