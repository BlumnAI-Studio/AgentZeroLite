# 수행 요약

WPF 셸의 7개 UI 결점을 한 번에 정리. 미션 브리프대로 펜슬 디자인을 먼저
작성하고 (`Docs/design/M0022-ui-refresh.pen`, 7개 BEFORE/AFTER 비교 프레임)
operator 확인 후 구현 진입. operator 가 직접 `Docs/design/agent-bot.png` 봇
아바타를 교체 편집해 채택, 이를 `.ico` + 인앱 PNG 리소스로 변환해 셸/탭바/
실행 아이콘에 일관 적용.

## 결과 (7개 변경 모두 반영)

1. **Application full-screen** — `WindowStyle="None"` + `WindowState.Maximized`
   이 작업표시줄까지 가리던 문제 수정. `SourceInitialized` 에서 P/Invoke
   `WM_GETMINMAXINFO` 훅을 걸어 maximize 영역을 `SystemParameters.WorkArea`
   에 클램프.
2. **AgentCLI → AgentBot 리네임 + 봇 아이콘** — ActivityBar 첫 번째 버튼
   툴팁/콘텐츠와 bottom-tab `AGENT CLI → AGENT BOT` 라벨 변경. 타이틀바
   다이아몬드 모티프도 같은 봇 PNG 로 교체. 내부 Akka 액터 경로
   `/user/stage/bot` 은 그대로 유지(M0013 컨벤션).
3. **WORKSPACES 완전접기** — 헤더 chevron 토글로 SidePanel 컬럼을 32-px
   레일로 축소 (헤더 라벨 / SidebarBody 가시성 collapse, splitter 폭 0).
   상태는 `AppWindowState.IsSidebarCollapsed` 에 영속화, 다음 실행 시 복원.
   EF 마이그레이션 `AddSidebarCollapsed` 자동 생성.
4. **`agentzero.ico` 교체** — `Docs/design/agent-bot.png` 의 1024×1024 RGBA
   소스를 PIL 로 `16/24/32/48/64/128/256` 다중 사이즈 `.ico` 와 256-px PNG
   리소스로 패킹. `csproj` 에 `Assets/agent-bot.png` Resource 등록.
5. **Scrap Detach / DockBack** — `FloatingWebDevWindow` 패턴을 차용한
   `FloatingScrapWindow` 신설. `ScrapPagePanel` 의 새 `DETACH` 버튼이
   `DetachRequested` 이벤트를 발생시키면 MainWindow 가 동일 UserControl
   인스턴스를 그대로 reparent (in-flight 캡처 상태 보존). 플로팅 창의
   ⤡ DockBack / X / 시스템 close 모두 원위치 복귀로 매핑.
6. **하네스뷰 — ActivityBar 4번째 아이콘** — `HarnessViewPanel` 신설
   (WebView2 lazy-init, 표시 시점에 `EnsureCoreWebView2Async` → URL
   navigate). WebDev/Scrap 와 동일 z-order(Grid.Column 1, ColumnSpan 3,
   Row 0, RowSpan 3). 툴바에 Reload / External 진입.
7. **Voice Note 확장 + 빈상태 제거** — NOTE 를 BotDockArea(30~150 px) 의
   협소 탭에서 빼내어 ActivityBar 5번째 아이콘 → `NotePage` 전체-오버레이
   로 승격. `OpenNoteTab()` 는 이제 `NotePage.Content` 로 reparent.
   `DocumentViewerPanel.txtEmpty` "Select a file to view" 빈상태 TextBlock
   제거 (요소는 Visibility=Collapsed 로 남겨 컴파일 호환).

## 후속 수정 (operator smoke-test 중)

### 후속 수정 #1 — 사이드바 접은 뒤 다시 펼칠 방법이 없음

operator 보고: "AgentCLI에서 접기한후 다시 펼치기 할 방법이 없음".

진단: 헤더 DockPanel 의 `Margin="12,0,8,8"` 이 좌우 20-px 를 잠식, 접힌
컬럼 폭 24-px 안에서 chevron 버튼이 사실상 0-px 로 렌더링되어 클릭 불가.

수정: `Project/AgentZeroWpf/UI/APP/MainWindow.xaml` —
`SidebarHeaderPanel` x:Name 부여 + chevron `MinWidth=28`.
`Project/AgentZeroWpf/UI/APP/MainWindow.xaml.cs.ApplySidebarCollapsed` —
접힘 시 헤더 마진을 `(0,4,0,4)` 로, 컬럼 폭을 24 → 32-px, chevron
`HorizontalAlignment.Stretch` 로 전환해 32-px 레일 전체가 클릭 표적이
되도록 변경. 펼침 복원 시에는 마진/얼라인을 원복.
빌드 재검증: `dotnet build … -c Debug` → 오류 0.

### 후속 수정 #2 — HarnessView/Note 에서 AgentBot 전환 불가

operator 보고: "대메뉴를 전환하면 우측에 전환됨.. 하지만 다른곳(하네스뷰등)
을 누른후 AgentBot을 누르면 전환이 안됨... AgentBot을 제외하고 나머지 메뉴
에서는 잘됨".

진단: `OnActivityBotClick` 의 overlay-닫기 체인이 M0019 시점 그대로
`CloseWebDev/Scrap/Settings` 만 호출. 이번 미션에서 새로 추가된
`HarnessViewPage / NotePage` 가 여전히 우측 전체를 덮고 있어, 봇 토글은
실행되지만 시각적으로 가려져 있어 "전환 안 됨" 으로 보임.

수정: `Project/AgentZeroWpf/UI/APP/MainWindow.xaml.cs.OnActivityBotClick`
에 `CloseHarnessView()` + `CloseNote()` 추가. 다른 진입점들은 이미 새
오버레이를 닫고 있어 별도 수정 불필요.
빌드 재검증: `dotnet build … -c Debug` → 오류 0. operator "확인 완료".
