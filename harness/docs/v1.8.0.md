# v1.8.0 — music-curator expert + Music LLM knowledge layer

**Released**: 2026-06-06
**Bump rationale**: Minor — first new agent since v1.7.0
(`test-runner`). Adds a 7th specialist (`music-curator`) for the
on-device music understanding subsystem that landed in the product
this cycle (Audio Spectrogram Transformer + WASAPI loopback +
realtime spectrum). No existing agent, engine, or knowledge file
is modified beyond the config registration.

## Why this change

AgentZero Lite shipped its first **audio understanding** feature this
cycle — the Music tab (Settings → 🎵 Music) runs MIT's
`ast-finetuned-audioset-10-10-0.4593` ONNX model on a sliding window
fed from either the microphone or any system playback endpoint via
WASAPI loopback. That introduced three concern areas not previously
present in the codebase:

1. **A classifier model with a strict input contract** — 1024×128
   log-mel with AST-specific normalization. Wrong preprocessing
   silently degrades accuracy without throwing any error.
2. **A WASAPI loopback capture path** — different sample rates, channel
   counts, format probing, MMDevice lifecycle than the existing mic-only
   `WaveInEvent` flow.
3. **Realtime UI** — sliding 10 s PCM buffer + 1.5 s inference cadence
   + 30 Hz spectrum repaint, all without blocking the dispatcher.

None of the existing specialists own those concerns cleanly:
- `code-coach` covers general C# / WPF / Akka but doesn't know AST
  preprocessing or AudioSet label conventions.
- `test-sentinel` owns voice round-trip testing (STT/TTS quality) but
  not multi-label classification or spectrum DSP.
- `security-guard` should still review the model download path, but the
  preprocessing fidelity / capture format / spectrum normalization
  questions are outside its scope.

Hence `music-curator` — a dedicated expert with the model card, capture
pipeline, and spectrum sensitivity conventions in one place.

## Rollback procedure

If `music-curator` proves redundant (e.g. AST is the only music model
forever and `code-coach` could just absorb the knowledge):

1. Revert `harness/harness.config.json` to v1.7.2.
2. Delete `harness/agents/music-curator.md`.
3. Move (don't delete) the three knowledge files to
   `harness/knowledge/code-coach/` and rename headers' owner field.
4. Update README pointers from "music-curator" to "code-coach".
5. Add a v1.8.1 doc explaining the consolidation.

The knowledge files themselves are the durable artifact — they survive
whichever agent ends up owning them. The agent definition is the
disposable wrapper.

## Forward signal

Tracked follow-ups that the music-curator should pick up next:

- **Kaldi parity preprocessing** — port `torchaudio.compliance.kaldi.fbank`
  to pure C# (Povey window + pre-emphasis + Slaney mel + energy floor)
  if Top-K drift on environmental sounds becomes user-visible.
- **MERT integration** — same shape as AST via `IMusicClassifier`;
  better for pitch / timbre tasks because it was trained with a CQT
  teacher on 160k hours of music.
- **CLAP integration** — text-conditioned zero-shot
  (`"solo violin"`, `"distorted electric guitar"`); same shape as AST
  plus an extra text prompt input.
- **Sensitivity slider** — single user-facing knob that shifts
  `floorDb` + `ceilDb` together (preserve range, slide window). Design
  pinned in `spectrum-sensitivity-conventions.md`.
- **GPU inference toggle** — add `Microsoft.ML.OnnxRuntime.DirectML`
  alongside the CPU package; expose `MusicSettings.UseGpu` once the
  installer size budget allows.
